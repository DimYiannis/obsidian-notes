# Big-O notation

Describes how runtime or memory grows as input size *n* grows — the worst-case growth rate, ignoring constants. Common classes, fastest to slowest: `O(1)` constant · `O(log n)` logarithmic · `O(n)` linear · `O(n log n)` · `O(n²)` quadratic.

Every structure here is a trade-off table of Big-O costs: [[array]] reads are `O(1)` but inserts `O(n)`; [[hash-table]] lookups average `O(1)`; [[binary-search-tree]] operations are `O(log n)` when balanced.
