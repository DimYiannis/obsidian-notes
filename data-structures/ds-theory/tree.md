# Tree

A hierarchy of [[node-pointer|nodes]]: one **root**, every other node has exactly one parent. No cycles — a tree is a connected [[graph]] with `n − 1` edges.

Vocabulary: **leaf** (no children) · **depth** (distance from root) · **height** (longest root-to-leaf path) · **subtree** (any node + its descendants — trees are recursively defined, see [[recursion]]).

Models anything nested: file systems, the DOM, org charts, parse trees. Most-used special case: [[binary-tree]].
