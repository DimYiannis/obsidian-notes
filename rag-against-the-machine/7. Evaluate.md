
Measure retrieval quality: recall@k with IoU-based span relevance. Own `evaluate` command reimplements this independently — never import or shell out to the moulinette. Theory: [[retrieval-evaluation]].

## Recall@k

Per question: is **at least one** of the top-k retrieved spans relevant? Yes/no, averaged over the dataset. One hit among k results scores the question; junk alongside it costs nothing.

## Relevance = IoU > 0.05

IoU on character ranges: `intersection / union` of retrieved span vs gold span.

```
spans (100, 300) and (250, 400):
intersection = 50, union = 300 → IoU ≈ 0.167  ✓ (> 0.05)
```

Low bar → right file + rough region suffices → this is why chunking biases large.

## Why recall, not precision

The generator can ignore junk chunks in the prompt; a **missing** relevant chunk is unrecoverable. RAG optimizes recall at retrieval, precision at generation.

## Targets

Recall@5 ≥ 80% docs / ≥ 50% code (aim 85/55 — private datasets may be harder).
