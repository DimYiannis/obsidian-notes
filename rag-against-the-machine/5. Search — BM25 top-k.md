

Query time: tokenize the query (same tokenizer as indexing), collect candidate chunks from the query terms' posting lists, score with BM25, take top-k. Theory: [[bm25]], [[lexical-retrieval]].

## The formula (whiteboard-ready)

```
score(D, Q) = Σ_{t ∈ Q}  IDF(t) · tf·(k1+1) / (tf + k1·(1 − b + b·|D|/avgdl))

IDF(t) = ln((N − df + 0.5) / (df + 0.5) + 1)
```

## The two knobs

- **k1** (~1.2–2.0): TF saturation. k1=0 → binary presence; large → raw counts. The `tf/(tf+k1·…)` form asymptotes — 50th occurrence adds ~nothing.
- **b** (0–1): length normalization. 0 → ignore length; 1 → full `|D|/avgdl` penalty for long chunks.

## Intuition in one line

TF-IDF + saturation + length normalization. Rare term (`lora`) decides the ranking; ubiquitous term (`self`) contributes ~nothing (IDF ≈ 0).

## Top-k

`heapq.nlargest(k, candidates)` — O(n log k), no full sort. ~30 lines total over the index.

## Tuning discipline

Every retrieval-affecting change (chunking, tokenizer, k1/b) gets measured: `search_dataset` + `evaluate` on both public datasets, recall@5 before/after. No blind tuning.
