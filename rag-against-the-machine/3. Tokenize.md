
Turn chunk text (and later, queries) into match terms. Theory: [[retrieval-tokenization]] — this is lexical analysis for search, **not** LLM BPE tokenization.

## Contract

1. Lowercase
2. Split on non-alphanumerics
3. Split `snake_case` and `CamelCase` into subtokens
4. **Keep the original identifier as a token too**

```
enable_lora  →  enable_lora, enable, lora
OpenAIServingChat  →  openaiservingchat, openai, serving, chat
```

## Why keep the original

Natural-language queries ("how to enable LoRA") hit the subtokens; exact-identifier queries (`enable_lora`) hit the intact term with its full high-IDF weight. This is the main lever for the 50% code-recall target.

## The iron rule

**Same tokenizer for indexing and queries — always.** Different analyzers = vocabulary mismatch = zero term overlap = zero recall, silently.
