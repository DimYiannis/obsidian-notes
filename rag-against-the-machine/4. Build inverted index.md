
One pass over all chunks: tokenize, append to posting lists. Built once (offline), queried many times. Theory: [[inverted-index]].

## Structure

```
index:        term → [(chunk_id, tf), ...]     # posting lists
chunk_meta:   chunk_id → (file_path, first, last, length)
stats:        N (chunk count), avgdl (average chunk length in terms)
```

`chunk_meta` lengths + `avgdl` feed BM25's length normalization. `df` per term = posting list length.

## Why this makes search fast

A query only touches the posting lists of its own terms. Chunks sharing **no** term with the query are never visited, never scored. Corpus size stops mattering; posting-list length is what counts. This is how 200 queries finish in < 10 s.

## Persist

Serialize index + metadata to `data/processed/` (path always a CLI argument, never hard-coded). `index --max_chunk_size 2000` rebuilds from scratch. Budget: whole corpus ≤ 5 min, target < 1 min.
