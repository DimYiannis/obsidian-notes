# RAG Against the Machine — Project Overview

Retrieval-Augmented Generation system over the vLLM 0.10.1 codebase: BM25 lexical retrieval over a hand-rolled inverted index, answer generation with Qwen3-0.6B, graded on recall@k by an external binary (moulinette), defended at a whiteboard.

## The problem

Qwen3-0.6B has never seen vLLM 0.10.1 internals — its parametric knowledge can't answer questions about this codebase. Pasting the whole corpus into the prompt is impossible: context window is finite.

## The solution

Retrieve first, generate second. Index the corpus once; per question, retrieve the k most relevant chunks and place them in the prompt so the model answers from context ([[grounding]]), not memory.

## Two pipelines

- **Indexing (offline, once)**: corpus → chunk → tokenize → inverted index
- **Query (per question)**: query → tokenize (same tokenizer!) → BM25 top-k → prompt → generate → evaluate

## Targets

| Metric | Requirement | Aim |
|---|---|---|
| Recall@5 (docs) | ≥ 80% | 85% |
| Recall@5 (code) | ≥ 50% | 55% |
| Indexing time | ≤ 5 min | < 1 min |
| 200 queries | ≤ 90 s | < 10 s |

## Stack

Python 3.10+ · uv · Python Fire CLI · pydantic · Qwen3-0.6B · HuggingFace
