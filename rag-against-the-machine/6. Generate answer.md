
Build a prompt from the question + top-k retrieved chunks, generate with Qwen3-0.6B. Theory: [[grounding]], [[lost-in-the-middle]].

## Prompt construction

- Clear chunk delimitation (source path + fenced text per chunk)
- Explicit instruction: answer **from the context below**; say so if the context doesn't contain the answer
- In-context learning — no fine-tuning anywhere

## Why grounding carries everything

Qwen3-0.6B has ~zero parametric knowledge of vLLM internals. If the right chunk isn't in the prompt, the answer is lost — retrieval recall is the binding constraint, generation can't recover it.

## Chunk ordering

Models attend worst to the middle of long context (U-curve). Put highest-scored chunks at the edges — top-ranked first. Small models suffer more; worth testing empirically.

## Mechanics

HuggingFace transformers, chat template, respect the context window (chunk size × k must fit). `Qwen/Qwen3-0.6B` must work; other models optional on top.
