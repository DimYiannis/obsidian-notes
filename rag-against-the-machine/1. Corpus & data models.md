
The corpus is the vLLM 0.10.1 source tree at `data/raw/vllm-0.10.1/`.

## What gets indexed

Everything textual: `.py`, `.md`, `.rst`, `.txt`, configs. Skip binaries/images. **Don't exclude tests** — questions may target them.

## Ground rules

- `file_path` in every output is compared **verbatim** to the reference: relative to project root, exactly as in the corpus (`data/raw/vllm-0.10.1/docs/features/lora.md`). Never absolute, never re-normalized.
- Read each file **once** as a single UTF-8 string. All offsets are computed against that exact string.

## Data models (pydantic, names exact)

MinimalSource · UnansweredQuestion · AnsweredQuestion · RagDataset · MinimalSearchResults · MinimalAnswer · StudentSearchResults · StudentSearchResultsAndAnswer

- A pydantic validator enforces `last - first <= 2000` — an invalid MinimalSource **cannot be constructed**. One violation invalidates an entire output file.
- Dataset JSON may hold answered or unanswered questions (union type) — parse via RagDataset, don't assume fields.
