## Definition

The standard scoring function for [[lexical-retrieval]], from the probabilistic relevance framework (Robertson & Zaragoza). BM25 is [[tf-idf]] evolved: it keeps IDF but adds **TF saturation** (parameter k1) and **document length normalization** (parameter b).

For a query Q and document D:

```
score(D, Q) = Σ_{t ∈ Q}  IDF(t) · ( tf · (k1 + 1) ) / ( tf + k1 · (1 − b + b · |D|/avgdl) )

IDF(t) = ln( (N − df + 0.5) / (df + 0.5) + 1 )
```

where `tf` = term frequency of t in D, `N` = number of documents, `df` = documents containing t, `|D|` = document length, `avgdl` = average document length.

## Key Properties

- **k1 — TF saturation** (typical 1.2–2.0): controls how quickly repeated occurrences stop adding score. k1 = 0 → binary presence/absence; large k1 → raw counts dominate. The `tf/(tf + k1·…)` form asymptotes, so the 50th occurrence adds almost nothing.
- **b — length normalization** (0–1): b = 0 → document length ignored; b = 1 → full penalty proportional to `|D|/avgdl`. Compensates for long documents matching more terms by chance.
- **IDF with +0.5 smoothing** — rare terms get high weight; the smoothed form avoids division by zero and negative weights.
- **Bag-of-words** — inherits the no-word-order assumption from [[tf-idf]].
- **Served by an [[inverted-index]]** — only documents sharing ≥1 query term need scoring.

## Examples

- *RAG Against the Machine*: hand-rolled BM25 (~30 lines over an inverted index) scoring code/doc chunks from the [[vllm]] corpus; chunk-size tuning interacts with b (see [[chunking]]).
- Lucene/Elasticsearch default similarity since 2016.
