# Call me maybe — Project Overview

Function calling system that translates natural language prompts into structured, schema-validated JSON function calls using a 0.6B parameter model (Qwen3-0.6B) and constrained decoding.

## The problem

A small LLM left to its own devices produces valid JSON ~30% of the time. We need 100%.

## The solution

Constrained decoding — at every token generation step, mask out logits for anything that would violate the JSON schema before the model picks the next token. The model handles meaning; the constraint layer handles structure.

## Two guarantees enforced simultaneously

- **Syntactic**: valid JSON structure throughout
- **Semantic**: function name must be from the known set; argument keys and types must match the schema

## Result

| Approach | JSON validity |
|---|---|
| Unconstrained | ~30% |
| Constrained decoding | 100% |

Function selection accuracy: ~90%+ (model-dependent, not structure-dependent).

## Key design decisions

- Greedy decoding for constrained steps — once invalid tokens are masked, sampling adds noise without benefit
- Pydantic for all input/output validation
- Vocabulary mapping (token ID → string) enables prefix-matching against known function names
- Function selection via the LLM — no keyword matching or heuristics

## Stack

Python · Qwen3-0.6B · HuggingFace · Pydantic
