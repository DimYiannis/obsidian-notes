When the state machine reaches `IN_NAME_VALUE`, the only tokens allowed are ones that spell out a valid function name from the definitions file.

The LLM's own probability distribution naturally favours `fn_add_numbers` because that's what makes sense given the prompt — the constraint just ensures it can't accidentally emit an invalid name.

Challenge: `fn_add_numbers` is likely split across multiple tokens by BPE — e.g. `fn`, `_add`, `_numbers`. The constraint must allow partial valid prefixes at each intermediate step, not just the complete name. This is prefix matching (`vocabulary.py`): at each step, allow any token whose string is a valid continuation of a known function name given what's already been emitted.
