Build a system prompt that gives the model everything it needs to make a decision.

```
Here are the available functions: fn_add_numbers (adds two numbers),
fn_greet (greets a person), fn_reverse_string (reverses a string).
Given this request: 'What is the sum of 40 and 2?', output a JSON function call.
```

This text includes:
- All available function names and descriptions (from `functions_definition.json`)
- The user's original prompt
- An instruction to output a JSON function call

The model never executes the function. It only generates a structured description of which function to call and with what arguments. Execution happens in the application layer.
