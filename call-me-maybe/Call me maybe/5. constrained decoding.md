At every single token generation step, before the LLM picks the next token, look at its logits and mask out anything invalid.

The state machine (`constraints.py`) tracks exactly where we are in the JSON output:

```
START               → expect {
EXPECT_KEY          → expect "name" or "parameters"
IN_NAME_VALUE       → expect one of the known function names
EXPECT_COLON        → expect :
IN_PARAMETERS       → expect { then argument keys
IN_ARG_KEY          → expect one of the known arg names for chosen function
IN_ARG_VALUE        → expect value of correct type (string / number / boolean)
EXPECT_COMMA_OR_END → expect , or }
DONE
```

At each state, `vocabulary.py` finds which token IDs are valid continuations. All others get set to `-inf` before the model selects the next token. After selection, the state advances.

This is not prompt engineering. The mask is applied at the logit level — invalid tokens are structurally impossible to emit, not just unlikely.
